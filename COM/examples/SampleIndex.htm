Navisworks COM API Examples

Included here is all currently available examples of the Navisworks COM API.

Automation

  • AutoLaunchExample: Basic Automation - C# Application
    This is a simple application that shows basic launching of Navisworks through API automation and then have it load a model. It also demonstartes how to make sure correct copy of roamer is registered if there are side by side installs.
  • AutoSelectionExample: Manipulating Selections - C# Application
    This application demonstrates how to keep track of objects selected in the Navisworks interface and manipulate the selection. (e.g. making objects hidden or changing colour)
  • AutoClashExample: Automatic Clash - C# Application
    A demonstration of automating the process of running pre-defined clash tests in a model and generating a report of the results.
  • AutoSimpleScriptExample: Script Automation - VB Script
    This script file shows how to run Navisworks automation through simple VB Script files.
  • AutoUserPropsExample: User Defined Properties - C# Application
    Example of using API automation to add user defined properies to Navisworks.
  • AutoDrivePluginExample: Driving IO Plugins - C# Application
    Example of using API automation drive IO Plugins.
  • AutoPublishScriptExample: Publishing Files via Script Automation - VB Script
    This script file shows how to publish files using VB Script.

Plug-ins

Before attempting to use plug-ins, make sure to read these included notes

  • PluginClashRuleExample: Clash Ignore Rule - C# COM DLL
    Example of using a plugin to define a custom clash detective ignore rule. This plugin allows you to specify that any clashes should be ignored when found between two user defined layers.
  • PluginExportExample: Exporter - C# COM DLL
    A set of two export plugin examples, one showing how to export a list of viewpoint names into a CSV file, the other generating a HTML report of Clash Detective results.
  • PluginContextMenuLauncherExample: Context Menu Launcher - C# COM DLL
    A very simple plugin that adds an extra item to the Navisworks context menu when right-clicking on an object in Roamer. This extra option allows you to launch Notepad automatically.
  • PluginPropertyDatabaseExample: Property Database Link - C# COM DLL
    A property plugin that reads data from a seperate database and shows that in the Navisworks property dialog. To use this example the included readme file is essential reading.
  • PluginSimpleExample: Simple Plugin - C# COM DLL
    Simplest plugin that can act as boiler plate code for plugin development.

ActiveX

Note: On 64bit OS these examples wont build unless you have manually registered a dummy 32bit version of the Integrated ActiveX controls. See 'COM Interface.pdf' for more details.
  • ActiveXSnapShotExample: Snapshot - C# Application
    An embedded ActiveX control allows the user to navigate around a model taking "snap shots" of specific views that can be viewed later or saved as BMP images.
  • ActiveXLocaleSwitcherExample: Locale Switcher - C# Application
    Simple application showing how to switch between languages of the menus in the ActiveX control.
  • ActiveXFindExample: Find - C# Application
    An example of how to use the powerful but tricky find function, through the ActiveX control.
  • ActiveXDumpStructureExample: Dump Structure - C# Application
    This application uses the ActiveX control to produce a report of the entire structure of the loaded model.
  • ActiveXSimpleExample: Simple ActiveX Usage - C# Application
    This application shows simple usage of the ActiveX control embedded inside a C# WinForms .NET application.
